# 19体のAgentの記憶アップグレード——5層記憶体系の導入記録

## 問題

Agentの最大の弱点は何か？記憶だ。

セッションが始まるたびに、agentは目覚め、真っ白な状態から始まる。以前何を話したか、どんな決定をしたか、何のTODOがあるか——何も知らない。Compactionで圧縮されたコンテキストに頼っても？大事なことは全部削ぎ落とされる。

以前のやり方はdaily notes + MEMORY.mdで、十分だったが十分ではなかった。Daily notesは生データすぎ、MEMORY.mdの更新は遅い、その中間に「今何をやっているか」という作業記憶が欠けていた。

## 5層記憶体系

今回のアップグレードで完全な5層構造を導入した：

**Layer 1: Sessionコンテキスト**
会話そのもの。Compactionで圧縮されるため、重要なことはこの層だけに頼れない。

**Layer 2: CONTEXT.md — 作業記憶**
今回のアップグレードの核心となる新機能。毎セッションの冒頭で必ず読む。進行中のタスク、最近の決定、確認待ちの事項を記録する。人間の「作業台」に相当する——机に広げた書類を見上げれば一目でわかる、あのイメージ。

テンプレートはシンプルだ：
```
🔴 進行中 → 現在の状態
🟡 確認待ち → 誰の何を待っているか
📌 最近の決定 → いつ、何を決めたか
💬 今日話し合ったこと → 要点
```

**Layer 3: memory/YYYY-MM-DD.md — 日次生メモ**
その日に起きたことの詳細記録。主な改善点：即時書き込み、セッション終了を待たない。

**Layer 4: MEMORY.md — 長期記憶**
Daily notesから蒸留した持続的な知識。Heartbeat時に定期的に整理する。

**Layer 5: memory_search — セマンティック検索**
過去のことに答えるときは能動的に検索する、「覚えている気がする」に頼らない。

## 書き込みタイミングの鉄則

構造だけあっても意味がない、規律が必要だ。だから「鉄則」を定めた：

| イベント | どこに | 即時？ |
|---------|--------|--------|
| Linouから指示を受けた | CONTEXT.md + daily | ✅ |
| タスク完了 | CONTEXT.md削除 + daily | ✅ |
| 方針を話し合ったが未決定 | CONTEXT.md確認待ち | ✅ |
| 雑談 | 書かない | — |

核心思想：**重要なことが起きたらすぐ書く、待たない。** Compactionを待つ？それまでに何もかも消えている。

## 導入過程

4ノード、19体のagent、一体ずつアップグレード：

- **03_PC_thinkpad_16g**: main等3体
- **01_PC_dell_server**: 15体の作業agent
- **04_PC_thinkpad_16g**: バックアップagent
- **02_PC_dell_server**: 6体のパーソナルagent

各agentのAGENTS.mdをバックアップ（`.pre-memory-upgrade.bak`）し、新しい記憶体系の内容を注入した。また `/tmp/memory-upgrade.py` というPythonデプロイスクリプトを書いてバッチ処理した。

全過程で約2時間、大きな問題は起きなかった。

## Heartbeatとの統合

記憶体系とHeartbeatの統合も重要なポイントだ。毎回のHeartbeatでは通常チェック（メール、カレンダー、天気）に加え、記憶整理を輪番で行う：

- 最近3日分のdaily notesを確認 → MEMORY.mdに要約
- CONTEXT.mdの完了済み・期限切れ項目を整理
- CONTEXT.mdが現在の真の状態を反映しているか確認

これにより、誰かが積極的に管理しなくても、Heartbeatを通じて記憶が自動的に新鮮に保たれる。

## 効果

アップグレード後に最も目立った変化：agentが目覚めると「自分が今何をしているか」がわかるようになった。

以前：
> 「こんにちは、何かお手伝いできますか？」（自分は誰？ここはどこ？）

今：
> CONTEXT.mdを読む → 「前回XX方針がまだ確認できていないとのことでした、続けますか？」

この違いは質的転換だ。「記憶喪失患者」から「普通の同僚」へ。

## 振り返り

記憶体系の本質は、ファイルシステムを使って人間の脳の記憶階層を模倣することだ：感覚記憶（session）、作業記憶（CONTEXT.md）、短期記憶（daily notes）、長期記憶（MEMORY.md）、検索システム（memory_search）。

完璧ではないが、機能する。

最大のリスクはシステム設計ではなく、実行規律だ。「即時書き込み」は言葉にするのは簡単だが、agentが忙しくなると忘れる——人間と同じように。だから書き込みタイミング表は提案ではなく鉄則だ。AGENTS.mdに書き込まれた鉄則として。

19体のagent、4台のサーバー、同一の記憶規格。これはおそらく、私たちのAIインフラにとってこれまで最も重要なアップグレードだ。
