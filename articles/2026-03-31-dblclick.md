# ダブルクリックでインラインタスク追加 — UXを壊さない「触れる場所に入力欄を出す」実装

ホームラボのダッシュボードに、プロジェクト管理のカードビューを実装した。タスクを追加するとき、最初は「+ 追加」ボタンをカードの下に置いていた。それで動くには動くが、使っていて気になることがあった。**ボタンを探してクリックする、という動作自体が認知コストになる**。

「カードをダブルクリックしたら入力欄が出れば早い」と思った瞬間、実装してみた。

## 完成した動作

- プロジェクトカードのタスクリスト部分を**ダブルクリック**
- その場にインライン入力欄が展開される
- Enter で送信 / Esc でキャンセル / ✓ボタンでも送信
- 追加後は自動でリストの最下部にスクロール
- タスクが0件のときは「双击添加任务...」というヒントテキストを表示

Modal も別ページ遷移もない。**「今いる場所」で完結する。**

## 実装

イベントリスナーは `ondblclick` をカードのリスト部分に直接設定する。

```html
<div class="simple-task-list"
     id="tasklist-${project.id}"
     ondblclick="inlineAddTask(event, '${project.id}')">
```

関数本体は以下の通り。100行ちょっとだが、ほぼすべて「防衛コード」だ。

```javascript
function inlineAddTask(event, projectId) {
  // チェックボックス・削除ボタン・既存の入力欄はスルー
  if (event.target.closest(
    '.simple-task-item, .inline-add-task, .simple-task-checkbox, .simple-task-delete'
  )) return;

  const list = document.getElementById('tasklist-' + projectId);
  if (!list) return;

  // 既に入力中なら2重展開しない → focusだけ当て直す
  if (list.querySelector('.inline-add-task')) {
    list.querySelector('.inline-add-input').focus();
    return;
  }

  // ヒントテキストを一時非表示
  const hint = list.querySelector('.no-tasks-hint');
  if (hint) hint.style.display = 'none';

  // インライン入力エリアを動的生成
  const row = document.createElement('div');
  row.className = 'inline-add-task';
  row.innerHTML = `
    <input class="inline-add-input" type="text"
           placeholder="新任务..." maxlength="100">
    <button class="inline-add-confirm" title="确定(Enter)">✓</button>
    <button class="inline-add-cancel"  title="取消(Esc)">✕</button>
  `;
  list.appendChild(row);

  const input      = row.querySelector('.inline-add-input');
  const confirmBtn = row.querySelector('.inline-add-confirm');
  const cancelBtn  = row.querySelector('.inline-add-cancel');

  row.scrollIntoView({ block: 'nearest', behavior: 'smooth' });
  input.focus();

  async function submit() {
    const text = input.value.trim();
    if (!text) { cancel(); return; }
    confirmBtn.disabled = true;
    confirmBtn.textContent = '…';
    try {
      const r = await fetch('/api/simple-tasks/add', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ project: projectId, text })
      });
      const d = await r.json();
      if (d.ok) {
        await loadSimpleTasksData();
        renderSimpleProjects();
        setTimeout(() => {
          const newList = document.getElementById('tasklist-' + projectId);
          if (newList) newList.scrollTop = newList.scrollHeight;
        }, 50);
      } else {
        alert('添加失败: ' + (d.error || '未知错误'));
        cancel();
      }
    } catch(e) {
      alert('网络错误');
      cancel();
    }
  }

  function cancel() {
    row.remove();
    if (hint) hint.style.display = '';
  }

  input.addEventListener('keydown', e => {
    if (e.key === 'Enter')  { e.preventDefault(); submit(); }
    if (e.key === 'Escape') cancel();
  });
  confirmBtn.addEventListener('click', submit);
  cancelBtn.addEventListener('click', cancel);
}
```

## 設計で考えた3つのこと

### 1. `event.target.closest()` で既存UIと共存させる

ダブルクリックイベントはバブリングする。タスクのチェックボックスをダブルクリックしても入力欄が開いてしまうのは困る。`event.target.closest()` でクリック対象が「操作済みの要素」なら早期リターンする。これをサボると既存UIとの衝突がすぐ起きる。

### 2. 「入力欄が既に開いている」状態を正しく処理する

同じカードを連続ダブルクリックしたとき、入力欄が2枚重なって開くのは最悪の体験だ。`querySelector('.inline-add-task')` で存在チェックし、あれば `focus()` だけ当て直す。UIの二重生成は常に意識すべき落とし穴。

### 3. キャンセル後はヒントテキストを復元する

「双击添加任务...」というヒントはタスク0件のときだけ表示したい。入力欄を展開するときに `display: none` で隠し、キャンセルや追加完了後に `display: ''` で戻す。DOMを消さずに隠す方式にすれば、再描画しなくて済む。

## 副産物: スクロール挙動

`scrollIntoView({ block: 'nearest' })` はカードがビューポート内に収まっているなら何もしない、はみ出していれば最小限だけスクロールする。`block: 'start'` にするとページが激しく動いて驚かれる。`nearest` 一択。

追加後の `setTimeout(() => newList.scrollTop = newList.scrollHeight, 50)` は、`render()` が非同期で DOM を再構築するため、50ms 待ってから最下部にスクロールする。ここは正直「動いたから良し」に近い。render 完了のコールバックがあれば await できるが、Single-file SPA の限界でもある。

## まとめ

- **ダブルクリック = 「その場で編集」という強いUI慣習** を使う
- 防衛コードの3点セット: `closest()` で衝突回避 / 二重生成ガード / ヒントの復元
- `scrollIntoView({ block: 'nearest' })` は穏やかで良い
- コードは121行の diff で済んだ。Modal を作るより早かった

ホームラボのダッシュボードで毎日使っているが、気づいたら「+ ボタン」を押したことがなくなっていた。それが答えだと思う。
