## はじめに：なぜバックテストは良い結果なのに実際には損するのか？

クオンツ入門者の典型的なパターン：「良さそうな指標を発見」→「バックテストを書く」→「結果が好調」→「実際に取引して損失」。

**なぜこうなるのか？**

「感覚的に有効」と「統計的に有意」は別物だからだ。クオンツ因子研究には厳格な評価体系——IC、IR、Barraリスク中性化——がある。これをスキップすると、バックテストはデータ内のノイズを掘っているだけになる。

---

## 一、IC（情報係数）：単一因子有効性の定量指標

IC（Information Coefficient）= 当期の因子エクスポージャーと翌期の株式リターンの相関係数。

```python
import scipy.stats as stats

def calc_rank_ic(factor_series, return_series):
    """単期RankICの計算"""
    rank_factor = factor_series.rank()
    rank_return = return_series.rank()
    ic, _ = stats.spearmanr(rank_factor, rank_return)
    return ic
```

### 判断基準

| 指標 | 閾値 | 意味 |
|------|------|------|
| IC平均 | > 0.03（Pearson）/ > 0.05（RankIC） | 因子有効性の基本閾値 |
| IC正比率 | > 55% | 因子方向の安定性 |
| IC標準偏差 | 小さいほど良い | 因子安定性 |

IC = 0.05は相関がわずか5%——低く見えるが、中国株ではこれで注目に値する有効因子だ。

---

## 二、IR（情報比率）：有効性より安定性が重要

```
IR = IC平均 / IC標準偏差
```

- **IR > 0.5**：実戦価値の基本閾値
- **IR > 1.0**：優秀な因子

### 因子合成の推奨方法：**ICIR加重**

有効性と安定性を同時考慮する。各因子の重みをそのIRに比例させる方式。等重合成より科学的。

## 三、Barra CNE5/CNE6：中国株マルチファクターの「OS」

Barraモデルの核心的な役割：
1. **リスク帰属**：収益のどれだけが市場・業種・スタイルから来るか
2. **因子中性化**：アルファ因子に混入した業種/規模ノイズを除去
3. **ポートフォリオ最適化**：スタイル制御しながら超過収益を最大化

### CNE5の10スタイル因子

| 因子名 | 意味 |
|--------|------|
| BETA | 市場感応度 |
| MOMENTUM | モメンタム（525日、直近21日除く） |
| SIZE | ln(時価総額) |
| EARNYILD | 収益利回り |
| RESVOL | 残余ボラティリティ |
| GROWTH | 成長性 |
| BTOP | バリュー（純資産/時価総額） |
| LEVERAGE | 財務レバレッジ |
| LIQUIDTY | 流動性（売買回転） |
| SIZENL | 非線形規模 |

### CNE6の追加（2024年以降に適合）
- **Quality**（ROE安定性）
- **Sentiment**（アナリスト格付け変化）
- **Dividend Yield**（配当利回り）

## 四、因子研究の完全フロー

```
Step1: 因子構築（外れ値処理・標準化）
Step2: 因子中性化（業種・規模を回帰で排除）
Step3: 単因子検証（RankIC・ICIR・分位バックテスト）
Step4: マルチファクター合成（ICIR加重）
Step5: ポートフォリオ構築（業種制約・TE最小化）
Step6: バックテスト評価（超過収益・シャープ比・IR）
```

---

## 五、絶対に踏んではいけない5つの落とし穴

1. **業種中性化の未実施** → 同業種を大量選択、業種ローテーションで大幅ドローダウン
2. **外れ値未処理** → IC計算が歪む、3σまたはMADで処理必須
3. **先読みバイアス** → 財務報告の開示遅延を無視、公告日で整合すること
4. **取引コストの無視** → 中国株の実取引コストは両側約1.5%
5. **データマイニングバイアス** → 200因子テストして上位20を保留するのはノイズ。OOS検証必須
