# MattermostにVS Codeスタイルのファイルブラウザパネルを実装した話

## きっかけ

Mattermostで作業していると、チャット画面を開きながら同時にサーバーのファイル構造を確認したい場面がよくある。今まではMattermostのウィンドウとSSHターミナルを並べて使っていたが、行き来が面倒だった。

要望はシンプルだった。**右サイドパネルに、VS Codeスタイルのファイルツリーを表示したい。** デフォルトのThreadパネルをほぼ使っていないので、そこと入れ替える。

## 技術アプローチ

MattermostにはWebクライアントにReactコンポーネントを注入できるプラグインシステムがある。右サイドバー（Right-Hand Sidebar, RHS）は`registerRightHandSidebarComponent`でカスタムパネルを登録できる。

全体の構成はこうだ：

```
Mattermost Plugin (Go backend + React frontend)
  └── RHSパネル → ファイルツリーコンポーネント
        ├── SSH接続 → 対象サーバー
        └── ファイル一覧API → ツリー構造レンダリング
```

フロントエンドはReact + TypeScriptで実装し、Mattermost本体のデザイン言語に合わせた。

## ハマったポイント

### 1. スタイルの競合

プラグインのCSSはMattermostのテーマスタイルと競合しやすい。外部CSSファイルを使うとCSS詳細度の問題が何度も起きたため、最終的に**インラインスタイル**に切り替えた。すべてのスタイルをJSXの`style={{}}`に直接書き、CSSカスケードの問題を完全に回避する。

エレガントな解決策ではないが、安定している。

### 2. テーマへの追従

Mattermostはライトテーマとダークテーマの切り替えができる。色をハードコードするとパネルの背景が本体と合わなくなる。

解決策はMattermostが提供する`window.theme`オブジェクトを参照し、色を動的に生成することだ：

```typescript
const theme = window.theme || {};
const bgColor = theme.sidebarBg || '#1e1e1e';
const textColor = theme.sidebarText || '#d4d4d4';
```

これでパネルがMattermostのテーマに自動追従するようになった。

### 3. ファイルツリーのUIデザイン

VS Codeのファイルツリーには細かいこだわりがある：
- フォルダの展開・折りたたみは三角アイコン
- ファイル種別によるアイコン変化
- 選択状態のハイライト
- 右クリックメニュー（第2フェーズ予定）

まずコアの展開・折りたたみ機能を実装し、ファイルタイプのアイコンは絵文字で代用（シンプルさ優先）。右クリックメニューは次フェーズに持ち越した。

## 感想

既存のプロダクト（Mattermost）を拡張するのは、ゼロから作るより面白い。他者のアーキテクチャの隙間を探し、提供されたAPIを活用しながら、全体の体験を壊さないようにする必要がある。

VS Codeのファイルブラウザ自体が、数十年にわたるツリー型UIの進化の結晶だ。そのコアインタラクションを再実装してみて、細部のすべてに設計上の意思決定が込められていることを改めて実感した。

安定したら整理してオープンソース化する予定。
