Blog執筆は難しくない。難しいのは複数プラットフォームへの発行。コピペ、フォーマット調整、画像アップロード——AI Agentが最も得意とする繰り返し作業の排除です。

## プラットフォーム選定

| プラットフォーム | 接続方式 | 言語 |
|-----------------|---------|------|
| Zenn | GitHub連携 | 日本語 |
| Qiita | REST API | 日本語 |
| Hashnode | GraphQL API | 英語 |
| dev.to | REST API | 英語 |

**不採用**：Medium（API放置状態）、Note.com（API皆無）。**APIで発行できないなら自動化に含める価値なし。**

## Zenn：GitHub連携

`articles/`にMarkdownを置きGitHubにpush→Zenn自動同期。バージョン管理が自然に統合、ロールバックも容易。

## APIトークン永続化の教訓

当初トークンをAgentのセッションコンテキストに保存。セッション終了やAgent再起動でトークン消失。**セッション跨ぎの状態は必ずファイルまたはDBに永続化——コンテキスト記憶に依存してはならない。**

## バイリンガル発行フロー

中国語で執筆→日本語版(Zenn+Qiita)、英語版(Hashnode+dev.to)。翻訳は単純な言語変換ではなく、ターゲットプラットフォームの文化への適応が必要。日本語版はより丁寧な表現、技術用語は英語原文を保持。

## 発行パイプライン

1. 執筆（中国語Markdown）→ 2. 翻訳（日英2版）→ 3. フォーマット適応 → 4. 4プラットフォーム並行発行 → 5. 検証 → 6. 記録

執筆完了から全プラットフォーム公開まで通常5分以内。

## 所感

AI Agentの能力境界は「できるかどうか」ではなく「自動化する価値があるか」。APIを提供しないプラットフォームは諦める——自動化を受け入れるプラットフォームは十分にあります。
