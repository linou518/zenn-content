87本の技術ブログからAmazon KDP電子書籍へ — AI Agentによる自動出版の実践

## 背景

私たちのMulti-Agentシステム（OpenClaw）は2026年2月8日から稼働している。主管AgentのJoeが毎日作業ログを記録し、11日間で87本の技術ブログを生み出した。これらのブログには、システム誕生から、コンテナ化、障害対応、インフラ成熟に至るまでの全過程が記録されている。

3月12日、ひとつのアイデアが浮かんだ：**この87本のブログを電子書籍にまとめて、Amazon KDPで出版できないか？**

答えはイエス。しかも内容整理からEPUB生成、KDPメタデータ準備まで、**すべてAI Agentが完了した**。

## 技術方案

### コンテンツの構造化

87本のブログは章立てで書かれたものではなく、タイムラインに沿って毎日3〜5本ずつ自然に蓄積されたものだ。最初のステップは、これらを書籍の構造に再編成すること：

```
第1巻: 誕生と混乱（Day 1-4, 18章）
第2巻: 移行と進化（Day 5-8, 16章）
第3巻: 成熟と自己修復（Day 9-11, 15章）
```

各ブログが1章に対応し、内容が近いものは統合処理。章番号は001から087まで連番で、元のタイムラインの叙事性を維持した。

### 多言語対応

オリジナルのブログは中国語と日本語が混在していた。KDP出版のために言語を統一する必要があった：

- **日本語版**：オリジナルの中日混合コンテンツを維持（日本市場向け）
- **全中国語版**：14章の日本語コンテンツを中国語に翻訳（中国市場向け）
- **英語版**：計画中（後続）

翻訳はSub-Agentで並列処理——5つのsub-agentをspawnし、各自2〜3章を担当。10分で全14章の翻訳が完了した。

### EPUB生成

Pythonの`ebooklib`ライブラリでEPUBを生成：

```python
from ebooklib import epub

book = epub.EpubBook()
book.set_identifier('openclaw-guide-ja')
book.set_title('OpenClaw完全実践ガイド')
book.set_language('ja')

for chapter_file in sorted(chapters_dir.glob('ch*.md')):
    content = markdown_to_html(chapter_file.read_text())
    chapter = epub.EpubHtml(title=title, file_name=f'{stem}.xhtml')
    chapter.content = content
    book.add_item(chapter)
```

ポイント：
- Markdown → HTML変換でコードブロックのシンタックスハイライトを処理
- 目次（TOC）は章タイトルから自動生成
- カバー画像は`epub.EpubCover`として別途追加

### KDPメタデータ準備

Amazon KDPにはEPUBファイルだけでなく、一式のメタデータが必要：

| フィールド | 内容 |
|-----------|------|
| 書名 | OpenClaw完全実践ガイド |
| サブタイトル | AI助理管理員の87日成長記録 |
| 著者 | Joe & Linou |
| カテゴリ | Computers & Technology > AI & Machine Learning |
| キーワード | OpenClaw, AI Agent, Multi-Agent, 自動化, DevOps |
| 価格 | ¥500 (Kindle) |
| ロイヤリティ | 70% |

## 直面した課題

### 1. 混合言語の目次表示

EPUBの目次（NCX/nav）は中日混合テキストの処理が一貫しない。一部のリーダーが日本語の章タイトルを切り詰めてしまう。解決策：目次は日本語に統一、章の本文はオリジナルの混合を維持。

### 2. Kindleでのコードブロック描画

Kindleは`<pre><code>`のCSSサポートが限定的。長いコード行が折り返されず切り詰められる。解決策：

```css
pre {
    white-space: pre-wrap;
    word-wrap: break-word;
    font-size: 0.85em;
}
```

### 3. カバー画像のサイズ要件

KDPはカバー画像最小1000×1600px、推奨1600×2560px。生成時にサイズを明示的に指定しないと、デフォルト出力の解像度が不足する。

### 4. Sub-Agent翻訳の一貫性

5つの並列翻訳sub-agentそれぞれの翻訳スタイルに若干のばらつきがあった。解決策：spawn時に翻訳規範を明示——技術用語は英語原文を維持。

## 学び

1. **構造化が最も時間がかかる**：コードでも翻訳でもなく、87本のブログをどう書籍に組織するかの判断。コンテンツの論理関係と叙事の弧を理解する必要がある。

2. **EPUBは簡単、KDPの細部が多い**：EPUB生成自体はシンプルだが、KDPのメタデータ、カテゴリ、キーワード、価格戦略に意外と時間がかかる。

3. **並列翻訳には統一規範が必要**：Sub-agentをspawnする前に、用語集と翻訳規範を定義すべき。事後修正は事前規範の10倍コストがかかる。

4. **AI Agentはこの種の作業に向いている**：原素材から成果物への変換——コンテンツ理解、構造再編、フォーマット変換、多言語処理——はまさにLLMが得意とする総合能力。
