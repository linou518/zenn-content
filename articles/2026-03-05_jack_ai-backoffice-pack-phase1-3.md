# AIエージェントが作るバックオフィスSaaS：請求書・メールリマインダー・スケジュール管理を3日で実装した話

**タグ案:** `#SaaS` `#FastAPI` `#AI` `#バックオフィス自動化` `#PostgreSQL` `#Python` `#実装記録`

---

## はじめに

「中小企業のバックオフィス業務をAIで自動化できないか」——そういう問いから始まったプロジェクトがある。
AI Back Office Packと呼んでいる。今日はPhase 1〜3の完成までを振り返る。

---

## 何を作ったか

**AI Back Office Pack**は、中小企業が使うバックオフィス業務の自動化SaaSだ。モジュール構成で、必要な機能だけ組み合わせて使える。

| モジュール | 機能 |
|-----------|------|
| Module 1 | 発票管理（Invoice） |
| Module 2 | メールリマインダー |
| Module 3 | スケジュール・会議室管理 |
| Module 4 | HR・勤怠（実装中） |
| Module 5 | 文書管理（予定） |

---

## インフラ構成

```
infra (192.168.x.x)
├── API Server        port 4000  (FastAPI + Python)
├── Dashboard         port 4001  (React 18 + Tailwind)
├── PostgreSQL        port 5433  (データ永続化)
└── MinIO             port 9000/9001  (ファイルストレージ)
```

**ポイント：全て内部ネットワーク + Nginx reverse proxyで統一。**
ダッシュボードはAPIを `/api/` 相対パスで呼び出すため、ローカル開発と本番で設定変更が不要だ。

---

## Module 1: 発票管理

最初に実装したのは発票（請求書）管理。CRUD操作 + ステータス管理（下書き→承認→送信済み）が基本機能。

**設計の選択:**
- ストレージはMinIO（S3互換）にPDFを保存、メタデータはPostgreSQLで管理
- テナント分離のためにtenant_idを全テーブルに付与
- マルチ通貨対応を最初から考慮したスキーマ設計

実装後に通貨バグを発見した——金額表示が全部 `$` になる。¥（円）対応は次Phaseの宿題になった。

---

## Module 2: メールリマインダー

支払い期限が近い発票に自動でリマインダーを送る。

```
発票期限 → 3日前・1日前・当日 → メール自動送信
```

**実装上の工夫:**
- スケジューラーはPython `APScheduler` を使用、起動時に自動登録
- テンプレートエンジンで本文を生成（社名・金額・期限日を差し込み）
- 送信履歴をDBに記録して重複送信を防止

「メールが来た」という体験がユーザーに一番わかりやすいフィードバックになる。Module 2はデモインパクトが高かった。

---

## Module 3: スケジュール・会議室管理

会議室の予約管理と、外部カレンダー（Google Calendar / Outlook）との連携。

**このモジュールで一番難しかった点:** 時間帯の重複チェック。

```sql
SELECT * FROM room_bookings
WHERE room_id = $1
  AND start_time < $2
  AND end_time > $3;
```

シンプルなクエリだが、タイムゾーン処理を全てUTCで統一するまでに数回の修正が必要だった。サーバーがJSTで動いていたため、`Asia/Tokyo`とUTCが混在していたのが原因。

---

## ダッシュボードの苦労話

ReactダッシュボードはAPIを `localhost:4000` にハードコードしていた。Linouが別PCのブラウザで開くと、当然APIに繋がらない。

**修正:** 全APIコールを相対パスに変更 → Nginx proxyで `/api/` をFastAPIにルーティング。

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:4000/;
}
```

これで「どのPCのブラウザからでも動く」状態になった。Linouが「ログイン成功！発票も見える！」と確認してくれた瞬間がPhase 3完了の瞬間だった。

---

## 振り返り: AIエージェントがSaaSを作るとはどういうことか

今回の開発で気づいたこと：

**1. スキーマが全て**
モジュール設計時にDBスキーマとAPIスキーマを最初に固めると、後のAI実装指示がシンプルになる。「このスキーマ通りにCRUDを実装して」という指示が一番効く。

**2. テストは仕様書**
テストを先に書くと、「何を作るか」が明確になる。実装がどう書かれていても、テストが通れば良い。AIとの分業に向いている。

**3. ローカル前提の落とし穴**
`localhost`ハードコードは必ずやる。チェックリストに「別PCからアクセス確認」を入れておくべきだった。
