2010年代、モバイルAPIをいち早く構築した企業が、モバイルインターネット時代の入場券を手にしました。

2026年、同じロジックが繰り返されています——ただし今回の入場券は**MCPサーバー**と**A2Aインターフェース**と呼ばれています。

## はじめに：プロトコルがエコシステムを決める

AnthropicがMCP（Model Context Protocol）を2024年末にオープンソース化した時、多くの人は「AIツール呼び出し仕様」に過ぎないと思っていました。しかし2年後、MCPの月間ダウンロード数は**9,700万回**（2026年2月データ）を突破し、Microsoft、GitHub、Salesforce、AWSがSteering Committeeに参加しました。これはもはや一つのツールではなく、**次のインターネット層**の基礎インフラ標準です。

同時期、GoogleがA2A（Agent-to-Agent Protocol）を2025年4月に発表し、6月にはLinux Foundationへの管理移管を完了しました。速度之快、说明Google非常清楚：Agent間通信標準を制する者がAIエコシステムの神経系を制する。

---

## 三層スタック：Agentの世界のアーキテクチャ全体像

この二つのプロトコルをより大きなフレームワークで見ると、それぞれが異なる役割を担い、完全な三層スタックを構成していることがわかります。

```
┌─────────────────────────────────────────┐
│  AGENTS.md (プロジェクト層)              │
│  宣言：このプロジェクトにAgentが何個いて  │
│  何ができるか                            │
├─────────────────────────────────────────┤
│  MCP (ツール層)                          │
│  定義：AgentがどのようにAPIやデータを呼ぶか│
├─────────────────────────────────────────┤
│  A2A (通信層)                            │
│  規定：Agent同士がどのように通信・協調するか│
└─────────────────────────────────────────┘
```

この三層が重なって、「**Web 4.0インフラ**」と呼ばれるものが形成されます。

---

## MCPとは：ツール呼び出しの共通言語

MCPは三層アーキテクチャを定義しています。

- **Host**：LLMアプリケーション本体（Claude Desktop、OpenClawなど）
- **Client**：Host内部でMCPサーバーと通信する役割
- **Server**：ツール・データ・リソースを公開するサービス

**アナロジー**：MCPとAI Agentの関係は、REST APIとモバイルアプリの関係と同じです——呼び出し規約を統一し、エコシステムの活力を解放します。

### 2026年のMCPエコシステム規模

- 公開MCPサーバー数：**数万個**
- 月間ダウンロード数：**9,700万回**（2026.2）
- 主要参加者：Microsoft, GitHub, Cloudflare, Stripe, Linear

この成長速度は、Docker Hubの初期数年間を上回っています。

---

## A2Aとは：Agent間通信プロトコル

MCPが「AgentがどのようにツールをCallするか」を解決するなら、A2Aが解決するのは「Agent同士がどのように対話・協調するか」です。

### 三つの主流オーケストレーションパターン

| パターン | 特徴 | ユースケース |
|---------|------|------------|
| **Orchestrator（中央制御）** | 1つのメインAgentが複数のサブAgentを指揮 | タスクが明確、フロー固定 |
| **Router（インテリジェントルーティング）** | タスクタイプに応じて動的に分配 | 異種タスク、専門分業 |
| **Swarm（自律群）** | Agentが自律的に役割を交渉 | 複雑な問題、創造的探索 |

### なぜLinux Foundation管理が重要か

GoogleがA2AをLinux Foundationに移管したことは、このプロトコルが中立的な業界標準になることを意味します。歴史的に、中立機関が管理するプロトコル（HTTP, TCP/IP, Kubernetes）は最終的に市場を制しました。

---

## 二つのプロトコルの関係：競合ではなく補完

- MCPは**縦方向**：Agent ↔ ツール・データ
- A2Aは**横方向**：Agent ↔ Agent

HTTPとTCPのように、両方が必要です。
