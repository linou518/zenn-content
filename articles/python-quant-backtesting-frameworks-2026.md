## はじめに：クオンツ初心者がハマる罠

クオンツ投資に入門すると、必ず3つの名前に出会う：**Backtrader、VnPy、Qlib**。

「Backtraderが一番簡単」「VnPyが王道」「MicrosoftのQlibが未来」——こんな意見が飛び交い、どれを選ぶか迷い始める。

**でも、この問いの立て方自体が間違っている。**

この3つのフレームワークは根本的に異なる用途を持っており、互いに代替関係にない。それぞれが何を解決するのかを理解すれば、選択は自然に決まる。

---

## 一、3者の位置づけ：まず用途を整理する

| 項目 | Backtrader | VnPy | Qlib |
|------|-----------|------|------|
| **核心的位置づけ** | イベント駆動バックテストエンジン | フルスタック量化取引プラットフォーム | AIクオンツ研究プラットフォーム |
| **開発者** | 個人開発者 mementum | 上海量贝 / OSコミュニティ | Microsoft Asia Research |
| **主要言語** | Pure Python | Python + C++ | Python |
| **メンテナンス** | ⚠️ 停止（2019年以降） | ✅ 活発（VnPy 4.0 / Python 3.12対応） | ✅ 活発 |
| **GitHub Stars** | ~14,000 | ~24,000 | ~14,000 |
| **対象市場** | グローバル（米株・先物中心） | 中国株 + 先物 + 暗号資産 | グローバル（中国株公式データセットあり） |

最重要ポイント：**3つは異なるフェーズのツールであり、競合ではない。**

## 二、Backtrader：入門に最適、だが致命的な弱点あり

Backtraderの設計美学は優れている。`Cerebro`オブジェクトが戦略・データソース・アナライザーを管理し、抽象化が明確だ。30行のコードで完全な戦略とチャートを生成できる：

```python
class SmaCross(bt.SignalStrategy):
    def __init__(self):
        sma1, sma2 = bt.ind.SMA(period=10), bt.ind.SMA(period=30)
        self.signal_add(bt.SIGNAL_LONG, bt.ind.CrossOver(sma1, sma2))

cerebro = bt.Cerebro()
cerebro.addstrategy(SmaCross)
cerebro.run()
cerebro.plot()  # 1行でチャート出力
```

**強み：**
- 100以上のテクニカル指標を内蔵、TA-Lib統合対応
- ドキュメント品質が高く、コミュニティのQ&Aアーカイブが豊富
- 学習曲線が低く「イベント駆動バックテスト」の概念理解に最適

**致命的な問題：**
- **2019年以降メンテナンス停止**、Python 3.10+で互換性問題が頻発
- **シングルスレッド制限**、大量データのバックテストが極めて遅い
- **中国株への適応が弱い**、CTPインターフェースが内蔵されていない

**結論：** フレームワーク設計思想の学習と戦略ロジックの検証（小規模データセット）に使う。

## 三、VnPy：中国株クオンツのデファクトスタンダード

VnPyは単なるバックテストフレームワークではなく、**フルスタック量化取引プラットフォーム**だ：

- **バックテスト→本番のシームレスな切り替え**：同じ戦略コードでバックテスト合格後にそのまま本番稼働
- **GUI操作バックテスト**：Jupyter不要、GUIから直接実行+パラメータ最適化
- **CTA特化設計**：ストップロス・利確・スリッページ・手数料が内蔵

**パフォーマンス比較（実測）：**
- VnPy vs DolphinDB：同等データ量（2万行分足）でVnPyは約5〜8倍遅い
- VnPy vs VectorBT：ベクトル化計算のVectorBTはSQLiteからの読み込みが6倍以上速い

**制限事項：**
- 単一銘柄バックテストのみ（マルチファクター銘柄選択は独自拡張が必要）
- ML統合が弱く、機械学習型戦略は得意ではない

**結論：** 中国株/先物CTA戦略開発の第一選択。バックテストから本番まで一貫して対応。

---

## 四、Qlib：バックテストフレームワークではなく、AIクオンツ研究システム

**Qlibは従来のバックテストフレームワークではない。**

その核心は**機械学習クオンツ研究パイプライン**だ：

```
Infrastructure → DataServer（Pandasより10倍速い時系列データエンジン）
Learning Framework → LSTM / LightGBM / TCN / Transformer 内蔵
Workflow → 特徴量エンジニアリング → Alpha → ポートフォリオ最適化 → 実行シミュレーション
```

**Qlibの独自能力：**
- **Alpha158/Alpha360ファクターライブラリ内蔵**（158/360個のテクニカルファクター）
- **MLモデルライブラリが完備**：LSTM、LightGBM、Temporal Fusion Transformer
- **Microsoftが継続投資**：2025年にLLM機能を統合

**制限事項：**
- 習得コストが非常に高い（ML基礎＋クオンツ基礎の両方が必要）
- CTAトレンドフォロー戦略には不向き
- 本番取引接続が弱い

**結論：** MLファクター研究、クロスセクション銘柄選択戦略に最適。

## まとめ：3択ではなく、3つの組み合わせ

推奨する実践ルート：

1. **入門フェーズ**：Backtraderでイベント駆動の仕組みを理解する（1〜2週間）
2. **研究フェーズ**：Qlibでファクター発掘とML戦略検証を行う
3. **本番準備**：VnPyでCTPに接続し、本番稼働の安定性をテストする

**1つだけ選ぶなら：**
- 中国株CTA → **VnPy**（エコシステムが最も充実）
- MLクオンツ → **Qlib**（Microsoft保証、ファクターライブラリが豊富）

クオンツ取引の真の参入障壁はフレームワーク選びではなく、戦略自体のAlphaにある。
