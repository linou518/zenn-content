## はじめに：バリュエーションはテック投資最大の落とし穴

テスラのP/Eは200倍超、NVIDIAは37倍でもアナリストは買い推奨、一方でP/Eが15倍の銘柄が下落し続ける——なぜか？

答えはシンプルだ：**テック株は単一指標で評価できない。成長段階によって使うツールが違う。**

2026年2月時点でS&P 500 ITセクター全体のP/Eは36.19倍——10年平均（24.92倍）を1.84標準偏差上回り、「Overvalued」ゾーンに入っている。Mag7（七大巨頭）の予想P/Eは高値の40倍超から約29倍に下がったが、内部分化は著しい。**「Mag7をまとめて買う」という戦略は2026年に入り有効性を失いつつある。**

本記事では、本当に必要な6つのバリュエーションフレームワークと、2026年の実際の使い方を解説する。

---

## 一、なぜテック株の評価はこんなに難しいのか？

三つの本質的な理由がある。

**利益が不安定。** 高成長期は大量の研究開発費や設備投資で純利益が低い、あるいはマイナス。P/Eが機能しない。

**成長率の差が極端。** AIチップ企業の成長率200%超 vs 成熟ソフト企業の5%。同じP/Eの数字は横比較できない。

**ビジネスモデルが多様。** SaaS、ハードウェア+サービス、広告+クラウドの混合——それぞれ「適正バリュエーション」の論理が異なる。

だからプロの投資家は**複数の指標を組み合わせて**使う。

---

## 二、6つのバリュエーションフレームワーク

### ① P/E（株価収益率）

```
P/E = 株価 ÷ 1株あたり利益（EPS）
```

**適用場面：** 利益が安定した成熟テック企業（Microsoft、Apple、Google）

**2026年の参考水準：**
| 指数/銘柄 | 予想P/E |
|----------|---------|
| S&P 500 ITセクター | 36.19倍 |
| Mag7全体 | 約29倍 |
| S&P 500全体 | 22倍 |

**Trailing vs Forward P/Eの使い分け：**
- Trailing（LTM）：過去12ヶ月の実績ベース、保守的で信頼性高い
- Forward（NTM）：今後12ヶ月のアナリスト予測ベース、成長期待を反映
- 高成長株はForwardを優先。両者の乖離が大きい＝市場の期待が高い（リスクも高い）

### ② PEG（株価収益成長率）

```
PEG = P/E ÷ 予想EPS年成長率（%）
```

成長株評価の核心ツール。「高いP/Eが本当に割高か」を判断できる。

- PEG < 1：成長率が評価を支える、相対的に割安
- PEG ≈ 1：適正評価
- PEG > 2：割高リスク

**事例：** NVIDIA（2026年2月）、P/E 37倍、成長率51%、PEG ≈ 0.72 → 高いP/Eでも成長が支える

### ③ P/S（株価売上高倍率）

```
P/S = 時価総額 ÷ 年間売上高
```

赤字SaaS・AIスタートアップなど、利益がない企業に使う。「1円の売上に何円を払うか」を見る。高成長企業は高いP/Sが許容されるが、Rule of 40と合わせて使う。

### ④ Rule of 40（40の法則）

```
Rule of 40 = 売上成長率（%） + EBITDAマージン（%）
```

**SaaS・クラウド企業評価のゴールドスタンダード。**

- ≥ 40点：合格。バリュエーションプレミアムが正当化される
- < 40点：不合格。よほど評価が低くない限り慎重に

40点以上のSaaS企業は平均121%のプレミアムで取引されるというデータがある。

### ⑤ EV/Revenue（企業価値÷売上高）

```
EV/Revenue = 企業価値（時価総額 + 純負債） ÷ 売上高
```

経験則：**EV/Revenue ≈ 成長率の1/3が公正価値に近い。** 成長率30%なら適正EV/Revenueは約10倍。

### ⑥ DCF（割引キャッシュフロー）

```
内在価値 = Σ（将来フリーキャッシュフロー ÷ (1 + WACC)^t）+ 残存価値
```

理論上最も厳密だが、前提への感応度が高い。WACC 1%の変動、ターミナル成長率1%の変動で評価額が30〜50%変わる。**「ゴミを入れればゴミが出る」の典型。** 安定したキャッシュフローを持つ成熟企業（Microsoft、Apple）に限って有効。

## 四、バリュエーションを使う4つの原則

**原則1：縦の比較を横の比較より優先する。**
同じ企業の現在のP/Eと歴史的平均値の比較が最も意味ある。Appleの31倍はApple自身の歴史で見て高いのか低いのか——これが先に来る問い。

**原則2：横比較は同じレーンで行う。**
NVIDIAとAMDの比較は意味があるが、NVIDIAとAppleの比較は意味が薄い。

**原則3：成長段階がツールを決める。**
- 初期・赤字 → P/S + Rule of 40
- 高成長期 → PEG + 予想P/E
- 成熟安定期 → 実績P/E + DCF

**原則4：バリュエーションだけでは買えない。カタリストが必要。**
安いことが上昇を保証しない。利益の変曲点、製品サイクルの転換、市場ナラティブの変化——これらが揃って初めて「割安」が機能する。

## まとめ：2026年の正しいアプローチ

ITセクター全体の評価が歴史的高水準にある今、最重要の判断は：**「まとめて買う」から「選んで買う」へ。**

Mag7の内部分化は明確だ。NVIDIAには成長の裏付けがある。Googleは相対的に合理的。Teslaはストーリーで動く。Metaは投資のROIを証明できていない。

2026年のMag7 ETF一括買いは、2023〜2024年に比べて合理性が低下している。

個人投資家にとって大事なのは精密な計算より「評価チェックリスト」を持つこと：
1. この企業はどの成長段階か？
2. どのフレームワークを使うべきか？
3. 歴史・同業種と比べて今の評価はどうか？
4. 現在の評価を支えるカタリストは何か、いつ顕在化するか？

**バリュエーションは答えではなく、より良い質問を引き出すためのツールだ。**
